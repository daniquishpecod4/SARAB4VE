# Política de Seguridad

SARA maneja **datos sensibles de personas en situación de vulnerabilidad**
(ubicación, condiciones médicas, contactos). Por eso la seguridad no es opcional:
es parte central del proyecto.

## 🔒 Cómo reportar una vulnerabilidad

**Por favor NO abras un issue público** para reportar fallos de seguridad. Un
issue público le muestra el problema a posibles atacantes antes de que podamos
arreglarlo.

En su lugar:

1. Escribe un correo a **danielquishpek@gmail.com** con el asunto
   `[SEGURIDAD] descripción breve`.
2. Incluye:
   - Qué encontraste y dónde.
   - Cómo reproducirlo (pasos).
   - El impacto que podría tener.
3. Te responderemos lo antes posible para coordinar la solución.

> Si GitHub tiene habilitado **"Private vulnerability reporting"** en este repo,
> también puedes usar la pestaña **Security → Report a vulnerability**.

## ⏱️ Qué esperar

- **Confirmación** de que recibimos tu reporte: lo antes posible.
- **Evaluación** del problema y un plan de solución.
- **Crédito** público a quien lo reportó (si así lo desea), una vez resuelto.

## 🛡️ Buenas prácticas para colaboradores

- Nunca subas archivos `.env`, llaves o contraseñas al repo.
- Nunca subas datos reales de personas a ejemplos o pruebas.
- Usa datos ficticios para desarrollo y demos.
- Cifra o hashea los datos personales sensibles antes de almacenarlos.

Gracias por ayudarnos a mantener a salvo a quienes confían en SARA. 🙏
